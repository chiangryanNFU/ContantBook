# ContantBook UI 優化前後對比

## 概述
本文檔展示 ContantBook 應用程式在 UI 優化前後的視覺變化對比。

---

## 🎨 改進前 (Before)

### 原始 UI 特徵：
- ❌ 按鈕無顏色區分，使用系統預設灰色樣式
- ❌ 按鈕尺寸較小，不夠醒目
- ❌ 表格無顏色變化，純白背景
- ❌ 表頭使用預設樣式，不夠突出
- ❌ 整體間距不足，元素排列緊湊
- ❌ 無視覺層次，所有元素看起來平淡
- ❌ 輸入欄位無焦點指示
- ❌ 缺少互動回饋（hover、pressed 效果）

### 主要問題：
1. **缺乏視覺引導**：所有按鈕看起來一樣，使用者難以快速識別功能
2. **可讀性不足**：表格資料密集，難以追蹤各行
3. **互動性弱**：無懸停效果，使用者不確定是否可點擊
4. **過時的外觀**：使用系統預設樣式，缺乏現代感

---

### 新 UI 特徵：

## 🎯 使用者體驗改進

### 1. 更快的決策速度
- **改進前**：需要閱讀按鈕文字才能確定功能
- **改進後**：通過顏色即可快速識別（綠=新增，紅=退出）

### 2. 更好的資料追蹤
- **改進前**：白色背景，眼睛容易跳行
- **改進後**：交替顏色，輕鬆追蹤每一行資料

### 3. 更清晰的狀態指示
- **改進前**：不確定哪個欄位正在編輯
- **改進後**：藍色邊框明確指示當前焦點

### 4. 更強的視覺回饋
- **改進前**：點擊按鈕無變化，不確定是否有反應
- **改進後**：懸停變色、按壓位移，明確的互動回饋

## 💡 設計思路

### Material Design 原則
1. **Material is the metaphor** - 使用陰影和層次模擬真實材質
2. **Bold, graphic, intentional** - 大膽的色彩和清晰的視覺層次
3. **Motion provides meaning** - 通過動態效果提供回饋

### 色彩心理學
- **綠色**：成長、新增、確認 → 用於「新增」
- **藍色**：信任、專業、資訊 → 用於「匯出」
- **橙色**：注意、警示 → 用於「匯入」（提醒資料會變更）
- **紅色**：停止、危險 → 用於「結束」（提醒程式會關閉）

### 可用性優先
- 足夠的按鈕尺寸（最小 100×40px）
- 充足的間距（不少於 15px）
- 高對比度（文字與背景）
- 清晰的視覺回饋

## 📝 技術實現摘要

所有改進通過在 `contantbook.ui` 檔案中添加 Qt StyleSheet（QSS）實現：

```cpp
<property name="styleSheet">
  <string notr="true">
    // 主視窗樣式
    QMainWindow { background-color: #f5f5f5; }
    
    // 按鈕樣式（包含 ID 選擇器）
    #pushButton { background-color: #4CAF50; }
    #pushButton_2 { background-color: #2196F3; }
    // ... 等
    
    // 表格樣式
    QTableWidget { /* 樣式定義 */ }
    
    // 輸入欄位樣式
    QLineEdit { /* 樣式定義 */ }
  </string>
</property>
```

**優點**：
- ✅ 不需修改 C++ 程式碼
- ✅ 樣式集中管理
- ✅ 易於維護和更新
- ✅ 支援即時預覽（在 Qt Designer 中）
